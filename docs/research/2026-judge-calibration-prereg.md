# Pre-registration — Are the study-series LLM judges actually measuring anything?

Registered 2026-07-13 (KST), before any calibration corpus row, judgment, or
deterministic score existed. Side study to the rewrite-efficacy program
(Studies 0–3); **fully separated from Study 3** — different corpus, different
artifact directory (`artifacts/judge-calibration-2026/`), no feedback into any
Study 3 analysis or its registration.

## Background

Every efficacy claim in the study series rests on the fixed 3-judge panel
(kimi-k2.5 / gpt-5.5 / grok-4.5) scoring 0–100 "AI-likeness". The panel's
agreement was gated (Krippendorff α), but its **discrimination** — does a high
score actually mean AI-authored — has never been measured against ground
truth, and single-judge swings (e.g. kimi 35 vs 92 on adjacent Study 3
documents) are unexplained: signal or noise?

## Research questions

- **RQ-C1 (discrimination):** per judge, how well does `ai_likeness` separate
  certain-human from certain-AI Korean documents?
- **RQ-C2 (stability):** how much does a judge's score move on repeat scoring
  of the identical document?
- **RQ-C3 (deterministic comparison):** does patina's deterministic stylometry
  score, on the same leakage-free corpus, match or beat the judges?

## Corpus (fresh, leakage-audited)

- **Human side (target 20, floor 12):** new Korean web documents collected at
  registration time from the vetted S1 publisher set (toss.tech, korea.kr,
  developer docs), using the S1 Arm-D document shape (≥3 paragraphs,
  1,200–4,000 chars, same paragraph filters). **Exclusions enforced by
  sha256 AND source-URL dedupe** against: the S1/S2/S3 corpora, all
  `artifacts/rebaseline-2025/private/*` intakes (incl. HAP-E, KatFish,
  modern-generation stores), and `tests/fixtures/suspect-zones/**`. This
  guarantees the deterministic layer was never tuned, swept, or benchmarked
  on any row (the rebaseline threshold sweeps and KO-diagnostic calibrations
  all draw from those excluded stores).
- **AI side (24):** topic-paired generations from the collected human
  documents' public titles (title + register + length band only — the human
  text never enters a prompt; same pairing rule as S1). Family rotation
  gpt-5.5 / kimi-k2.5 / grok-4.5, 8 documents each, deterministic assignment
  (sorted by human sample_id, round-robin). **claude is excluded** as a
  generator: it is Study 3's rewriter (quota priority) and using only the
  three judge families enables a clean cross-family design below. If fewer
  than 16 human docs are collected, AI-side pairing reuses titles round-robin
  (still never the text) to keep 24.
- Labels are certain by construction. Raw text stays gitignored under
  `artifacts/judge-calibration-2026/`; hash-only metadata is committable.

## Design

- **Main pass:** each judge scores every document once — same judge prompt,
  parser, retry policy (3 attempts), and CLI invocations as the S2/S3 runners,
  byte-identical. Document order is shuffled deterministically (seed 20260713).
- **Cross-family scoring rule (fixed):** a judge's headline metrics are
  computed on human docs + the 16 AI docs NOT generated by its own family.
  Self-family cells are reported separately as a self-preference check, never
  pooled into the headline.
- **Stability block:** 5 documents (deterministic pick: 3 AI + 2 human,
  lowest sha256 hex order within class) × 4 additional repeats per judge
  (5 scores total per judge-document).
- **Deterministic layer:** `scoreText` (prose-score, lang ko) `score` field on
  all documents — no repeats (deterministic by construction).

## Metrics (fixed)

Per judge (cross-family subset):
- **Accuracy** — fraction of correct `authorship` calls.
- **AUC** — (Cliff's δ(AI scores, human scores) + 1) / 2, with a seeded
  5,000-iteration bootstrap CI (seed 20260713) resampling documents.
- **Bias** — mean `ai_likeness` on human docs and on AI docs (direction of
  error mass).
- **Stability** — per-document SD across the 5 repeat scores; summary =
  median SD over the 5 stability documents.
- **Self-preference** — mean `ai_likeness` on self-family AI docs minus mean
  on cross-family AI docs (negative = judge flatters its own family).

Panel (2-of-3 quorum mean, as in the studies): AUC and accuracy on the
intersection where every judge is cross-family (docs from... a panel-scored
document always includes the generator's own family judge, so panel metrics
are computed on ALL AI docs with all three judges pooled, reported as the
operational panel figure with the self-family caveat noted).

Deterministic layer: AUC and hot-verdict accuracy on the identical
document set.

## Pass / demote criteria (fixed before data)

- **Judge PASS (주심단 유지):** cross-family AUC ≥ 0.75 AND median repeat
  SD ≤ 12.
- **Judge DEMOTE candidate:** cross-family AUC < 0.65 OR median repeat
  SD > 20 → a queue **decision** item proposing replacement/reweighting; no
  panel change ships from this study alone.
- Between the bands: judge stays with a WATCH note in the results doc.
- **Panel PASS:** pooled panel AUC ≥ 0.80.
- **Deterministic promotion review:** if the deterministic AUC ≥ (best single
  judge cross-family AUC − 0.05), file a queue **decision** proposing primary-
  judge promotion / panel augmentation. Promotion itself is an operator call.
- Study 3's scores remain interpretable under any outcome: this study grounds
  their credibility but never edits Study 3 data or its registration.

## Quota & scheduling discipline

- Runs only in the post-midnight window; **zero claude usage**; single
  sequential lane with ≥15 s spacing between judge calls; Study 3's judge
  lane (~1 call per ~8 min) has priority — if any judge CLI starts refusing,
  this study backs off 30 min (resumable by row, same fail-soft/prune
  practice as S2/S3).
- Budget: ≤ ~210 judge calls + 24 generation calls, none on claude.

## Outputs

- `docs/research/2026-judge-calibration.md` (results, all pre-set verdicts).
- Queue: results summary as kind=report; any demote/promotion proposal as a
  separate decision.

No corpus row, generation, judgment, or deterministic score existed when this
registration was committed.
