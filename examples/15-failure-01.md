# Pattern 15: 인라인 헤더 목록 — 실패 사례 (오탐)

## 입력 텍스트

> ### 요청 파라미터
>
> - **userId:** 사용자 고유 식별자 (string, 필수)
> - **token:** 인증 토큰 (string, 필수)
> - **locale:** 응답 언어 코드, 기본값 `ko` (string, 선택)
> - **limit:** 반환할 최대 항목 수, 기본값 20 (integer, 선택)

## 기대 출력

> (교정 없음 — 패턴 15가 발화되어서는 안 되는 텍스트)

## 적용 패턴

- Pattern 15 (인라인 헤더 목록): "**userId:** ~", "**token:** ~", "**locale:** ~", "**limit:** ~"로 "**레이블:** 설명" 형식이 4회 사용되어 표면상 발화 조건에 도달함.

## 판정

**실패 (오탐)** — 이 텍스트는 API 파라미터 문서이며, "**파라미터명:** 설명" 형식은 기술 문서의 표준 양식이다. 제외 조건("API 파라미터 표, 기술 명세서·사양표 등 레이블-설명 구조가 표준 양식인 참고 문서")에 정확히 해당한다. 파라미터명을 볼드로 구분하고 콜론 뒤에 타입과 필수/선택 여부를 기재하는 것은 개발 문서의 관례이므로 교정 대상이 아니다.
