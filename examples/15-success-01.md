# Pattern 15: 인라인 헤더 목록 — 성공 사례

## 입력 텍스트

> 이번 업데이트의 주요 변화는 다음과 같습니다.
>
> - **사용자 경험:** 새로운 온보딩 플로우를 도입하여 가입 전환율을 높였습니다.
> - **성능:** 캐시 알고리즘을 개선하여 페이지 로딩 속도를 40% 단축했습니다.
> - **보안:** 종단간 암호화를 적용하여 데이터 유출 위험을 최소화했습니다.
> - **접근성:** 스크린 리더 호환성을 강화하여 시각 장애인 사용자 지원을 확대했습니다.

## 기대 출력

> 이번 업데이트에서 네 가지가 바뀌었다. 가입 과정을 새로 만들어서 전환율이 올랐고, 캐시 방식을 손봐서 페이지가 40% 빨라졌다. 암호화를 종단간으로 확장했고, 스크린 리더 지원도 넓혔다.

## 적용 패턴

- Pattern 15 (인라인 헤더 목록): "**사용자 경험:** ~", "**성능:** ~", "**보안:** ~", "**접근성:** ~"로 "**레이블:** 설명" 형식이 4회 반복된다. 일반 업데이트 공지에서 이 형식은 AI의 전형적인 구조화 습관이다.

## 판정

**성공** — 발화 조건("같은 목록에 '**레이블:** 설명' 형식 항목이 2개 이상")을 4개로 초과한다. 업데이트 공지는 API 파라미터 표나 기술 명세서가 아닌 일반 산문 콘텍스트이므로 제외 조건에 해당하지 않는다. AI가 정보를 기계적으로 분류해 동일 포맷으로 나열하는 전형적인 패턴이다. 교정 후 볼드 레이블 구조를 걷어내고 자연스러운 서술형 문장으로 통합했다.
