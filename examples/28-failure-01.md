# Pattern 28: 불필요한 외래어 남발 — 실패 사례 (과교정)

## 입력 텍스트

> 이 서비스는 REST API를 통해 외부 클라이언트와 통신하며, 백엔드는 Node.js 서버에서 실행된다. 데이터는 PostgreSQL 데이터베이스에 저장되고, 프론트엔드는 React 프레임워크로 구현되어 있다.

## 기대 출력

> (교정 없음 — 패턴 28이 발화되어서는 안 되는 텍스트)

## 적용 패턴

- Pattern 28 (불필요한 외래어 남발): "API", "클라이언트", "백엔드", "서버", "데이터베이스", "프론트엔드", "프레임워크"가 등장.

## 판정

**실패 (과교정)** — 이 텍스트에 등장하는 외래어는 모두 IT/개발 분야에서 정착된 전문 용어로, Pattern 28의 제외 조건에 해당한다. "API"를 "응용프로그래밍인터페이스"로, "서버"를 "접속 컴퓨터"로, "프레임워크"를 "개발 틀"로 바꾸는 것은 오히려 기술 문서의 정확성을 해친다. 대체 가능성이 있어야 하는 Pattern 28의 대상은 한국어 관행에 이미 자리 잡은 IT 전문 용어가 아니라, 비즈니스 맥락에서 남용되는 경영 외래어다.
